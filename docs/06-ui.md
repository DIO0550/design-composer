# 06. UI・キャンバス操作

GUI 編集のスコープを規定する。編集の実体はすべて「ドキュメント（JSON）への変更」であり、GUI は core に対するクライアントの1つである（AI・外部エディタと対等）。

## 画面構成

Figma 型の3ペイン。

| ペイン | 内容 |
|---|---|
| 左 | ツリービュー（データモデルの木をそのまま表示）＋部品一覧 |
| 中央 | キャンバス。artboard を配列順に自動配置。ズーム / パンは非永続の view state |
| 右 | プロパティパネル。スキーマから自動生成（03-schema）。`enabledWhen` により表示を出し分ける |

- アプリ内 JSON ビューは**持たない**。JSON を見たい場合は外部エディタに委ねる（自動保存によりファイルは常に最新）

## 選択

- キャンバス上のクリックとツリービューの選択は双方向に連動する
- 選択状態は非永続（実行時のみ）。外部編集によるリロード後の選択復元は name によるベストエフォート

## キャンバス直接操作

Figma 寄りの直接操作を採用する。

- **移動（ドラッグ）**: 座標の変更ではなく**ツリー内の移動**として解釈する（02-data-model）。ドロップ先は「どの Box の何番目の子になるか」をハイライトで提示する
- **リサイズハンドル**: `widthMode` / `heightMode` が `fixed` のノードにのみ表示し、ドラッグで `width` / `height` を変更する。`hug` / `fill` ではハンドルを出さない（スキーマのモードと直接操作が一致する）
- **Text のインライン編集**: ダブルクリックで `content` をその場で編集する

## 編集操作の一覧

| 操作 | 内容 |
|---|---|
| 挿入 | プリミティブ（Box / Text）または部品インスタンス（ref）を選択位置に追加 |
| 削除 | 選択ノード（サブツリーごと）を削除 |
| 並べ替え | 同一親内の子順序の変更（ドラッグまたはツリービュー） |
| props 編集 | プロパティパネル経由。トークン参照 prop はトークン一覧からの選択式 |
| artboard 操作 | 追加・削除・配列順の並べ替え |
| tokens 編集 | 種別ごとの一覧・追加・変更・削除の UI を持つ |
| コピー & ペースト | サブツリー単位。**自動リネームで名前の一意性を保つ**（01-file-format） |
| undo / redo | メモリ内管理。適用は通常の編集として自動保存される（05-architecture） |

## 部品化・解除

### 部品化（Create Component）

- 選択サブツリーを `components` へ移動し、元の位置に参照ノード `{ name, ref }` を置く
- 操作時に部品名のみを入力させる。**publicProps 宣言（binding）はゼロで作成する**。宣言の追加は AI / JSON 編集で行う（宣言編集 UI は初期スコープ外）

### 解除（Detach）

- インスタンス（ref ノード）を、部品定義に overrides を適用した実ツリーへ展開して置き換える
- **overrides の焼き込み**: 展開結果は「定義 + そのインスタンスの overrides」を合成した状態とする（レンダリング用の合成処理を流用）
- **自動リネーム**: 展開される内部ノードの name は単一名前空間で衝突するため、コピー & ペーストと同じ自動リネーム規則を適用する
