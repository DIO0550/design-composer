# UI表示確認規約

UIを実装・変更した場合は、**必ず playwright-cli で実際の表示を確認**してから完了とすること。

## 手順

1. 開発サーバーを起動する(`pnpm dev`。ポート 14000)
2. playwright-cli で対象ページを開き、スクリーンショットを取得する
3. スクリーンショットを目視確認し、以下をチェックする:
   - レイアウト崩れ・要素の重なり・はみ出しがないこと
   - 実装した機能の表示が仕様通りであること
   - 操作後の状態変化(クリック・入力後)も同様に確認すること
4. 確認したスクリーンショットの取得コマンドと確認結果を作業ログに残すこと

## ルール

- 「コードを書いた・テストが通った」だけではUI実装の完了とみなさない
- 静的な初期表示だけでなく、主要なユーザー操作フロー(要素の追加 → 編集 → 結果の反映など)を通しで確認すること
- ビューポートはデスクトップ(1280x720 以上)を基準とする
